Que es git?

Git es una herramienta distribuida de gestión de código fuente y control de revisiones.

Git fue creado por Linus Torvalds para su uso en el desarrollo del kernel de Linux y otros proyectos de código abierto. Según una charla que dio para Google, estaba buscando un Source Control Management (SCM) que cumpliera tres criterios:

  1. Ser distribuido.
  2. Ser rápido.
  3. Generará exactamente lo que se ingresó, o se imprimirá un error.

Este artículo cubrirá los primeros pasos con Git y su uso general.

Configuración

Antes de contribuir a un proyecto, es imperativo establecer un nombre de usuario y un correo electrónico para cada usuario. Sustituya las referencias a "Larry" entre corchetes (corchetes y todo lo que hay entremedio, pero deje las comillas) en el siguiente ejemplo por un nombre de usuario personal y una dirección de correo electrónico:

user $ git config --global user.email "<larry@gentoo.org>"
user $ git config --global user.name "<Larry the cow>"

Local

Si solo habrá un usuario del proyecto, o al crear algo que se compartirá de forma distribuida, comience en la estación de trabajo local.

Si la intención es tener un servidor central que todos usen como servidor "oficial" (por ejemplo, GitHub), entonces podría ser más fácil crear un repositorio vacío allí.

La siguiente lista de comandos describirá cómo crear un repositorio en una estación de trabajo:

user $ cd ~/src
user $ mkdir hello
user $ cd hello
user $ touch readme.txt
user $ git init

El repositorio local ya ha sido creado.

Hagamos algunas ediciones:

user $ echo "Hello, world!" >> readme.txt

El nuevo archivo readme.txt debe agregarse (prepararse) antes de poder incluirse en el repositorio de git. Utilice los siguientes comandos para preparar el archivo y realizar la confirmación: 

user $ git add readme.txt 
user $ git commit -m "Added text to readme.txt"
